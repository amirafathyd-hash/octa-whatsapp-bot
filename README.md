# Octa WhatsApp Bot — خدمة إرسال الصور التلقائي على واتساب

سيرفر Node.js منفصل تمامًا عن `octa-food-backend`، بيستخدم `whatsapp-web.js`
(واتساب ويب حقيقي متصل برقمك) عشان يبعت صور تلقائي على رقم ثابت من غير ما حد
يضغط أي حاجة في واتساب يدوي.

## خطوات النشر على Railway

1. **اعمل مشروع جديد على Railway منفصل** (مش جوه نفس مشروع `octa-food-backend`):
   - New Project → Deploy from GitHub repo (لازم ترفع المجلد ده على ريبو جديد على GitHub الأول، اسمه مثلاً `octa-whatsapp-bot`)

2. **ضيف Volume** (مهم جدًا — بدونها هتحتاج تمسح QR من جديد كل مرة السيرفر يعيد تشغيل):
   - في إعدادات الـservice على Railway → Volumes → Add Volume
   - Mount path: `/data`

3. **ضيف Environment Variables**:
   - `WHATSAPP_DEFAULT_NUMBER` = `201002822524` (الرقم اللي هيستقبل الصور، بصيغة دولية بدون + أو 00)
   - `WHATSAPP_BOT_API_KEY` = أي نص عشوائي طويل تختاره إنت (مفتاح حماية، عشان حد تاني ميقدرش ينده على الـendpoint من برا)

4. **Deploy** — السيرفر هيشتغل، وبعد كذا ثانية افتح:
   ```
   https://<اسم-السيرفر-بتاعك>.up.railway.app/qr
   ```
   هيظهرلك كود QR. افتح واتساب على موبايلك → الإعدادات → الأجهزة المرتبطة → ربط جهاز → امسح الكود.

5. بعد المسح، الصفحة هتتغير تلقائي وهيظهر "واتساب متصل بالفعل ✅". من اللحظة دي السيرفر جاهز يستقبل ويبعت صور.

## ربطه بالباك إند الأساسي (Python)

في `octa-food-backend` على Railway، ضيف Environment Variable:
- `WHATSAPP_BOT_URL` = الرابط بتاع السيرفر ده (مثلاً `https://octa-whatsapp-bot-production.up.railway.app`)
- `WHATSAPP_BOT_API_KEY` = **نفس** القيمة اللي حطيتها فوق بالظبط

## ملحوظات مهمة

- ده بيستخدم نسخة واتساب ويب غير رسمية (مش WhatsApp Business API المعتمد من Meta)، يعني فيه احتمال (بسيط لو الاستخدام خفيف زي حالتك) إن واتساب يعمل Ban للرقم لو حسّ إنه بوت. الأفضل تستخدم رقم احتياطي مش رقمك الشخصي الأساسي.
- لو السيرفر اتقفل وفتح تاني والـVolume مش متظبطة صح، هتحتاج تمسح QR تاني.
- مفيش حد تاني يقدر يستخدم الـendpoint غير لو عنده نفس الـ`WHATSAPP_BOT_API_KEY`.
